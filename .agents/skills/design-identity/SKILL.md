# Design Identity Skill

Read `AGENTS.md` first.

This skill defines how to apply the project’s visual direction in real UI work.

## Core formula
Use:
- ShipMonk for structure, hierarchy, spacing, and proof
- Kak2c for specificity, operational clarity, and marketplace relevance

Do not copy either source literally.
Use them as directional references only.

## Target outcome
The final result should feel:
- premium but grounded
- structured but not cold
- practical, not abstract
- conversion-ready without looking pushy
- cleaner than Kak2c
- warmer and more operational than ShipMonk

## When to use this skill
Use this skill when the task involves:
- building a new landing page
- redesigning logistics or fulfillment pages
- improving hero, trust, proof, services, CTA, or form sections
- making the UI feel more premium, more practical, or more credible
- removing generic SaaS styling
- improving mobile presentation of business sections

## Workflow

### 1. Start from business structure
Define first:
- what the company does
- who the page is for
- what business result it promises
- what proof supports the promise
- what CTA should be primary

Do not start from decoration.

### 2. Match each section to the right asset
Default mapping:
- Hero → `hero-shipmonk-good.png` + messaging influence from `hero-kak2c-good.png`
- Header → `header-clean-business.png`
- Trust/logos → `trust-strip-logos-good.png`
- Services → `service-cards-operational-good.png`
- Benefits → `benefits-kak2c-good.png`
- Proof/metrics → `metrics-proof-good.png`
- CTA → `cta-section-good.png`
- Desktop form → `form-simple-good.png`
- Mobile hero → `mobile-hero-good.png`
- Mobile services → `mobile-services-good.png`
- Mobile form / CTA-form block → `mobile-form-good.png`

### 3. Apply the design rules
Always aim for:
- strong hierarchy
- generous spacing
- practical service language
- clear commercial communication
- restrained visuals
- high trust
- one obvious conversion path

### 4. Keep the style balanced
Every section should feel:
- commercially serious
- easy to scan
- visually calm
- operationally credible
- useful rather than decorative

## Section rules

### Hero
Use `hero-shipmonk-good.png` as the main structure reference.

Requirements:
- one strong headline
- one supporting paragraph
- one primary CTA
- optional secondary CTA
- one relevant visual
- optional compact proof row

Add Kak2c influence through:
- direct wording
- practical promise
- clear service specificity

Avoid:
- vague slogans
- too many CTAs
- decorative hero visuals with weak meaning
- generic startup gradients

### Header
Use `header-clean-business.png`.

Requirements:
- simple top-level navigation
- clean spacing
- clear primary CTA
- no crowded menu behavior

Avoid:
- overloaded nav bars
- too many equal-priority actions
- messy enterprise complexity without need

### Trust strip
Use `trust-strip-logos-good.png`.

Requirements:
- even spacing
- clean alignment
- calm credibility presentation
- no visual clutter

Use for:
- clients
- marketplaces
- partners
- integrations

### Services
Use `service-cards-operational-good.png`.

Requirements:
- concrete service names
- short supporting text
- clear business value
- scan-friendly card structure
- restrained icon use

Good framing:
- fulfillment for marketplaces
- storage and handling
- packaging and labeling
- shipping coordination
- returns processing
- integrations and reporting

Avoid:
- vague benefit-only cards
- fluffy wording
- oversized decorative icons

### Benefits
Use `benefits-kak2c-good.png`.

Requirements:
- explain value simply
- focus on practical relief, speed, control, or reliability
- keep copy direct
- make content easy to skim

Avoid:
- abstract branding language
- hype wording
- long explanatory paragraphs

### Metrics and proof
Use `metrics-proof-good.png`.

Requirements:
- short, bold numbers
- minimal supporting copy
- realistic trust indicators
- clean grouping

Use proof early when possible.

Avoid:
- fake vanity metrics
- cluttered stat layouts
- oversized testimonial walls

### CTA
Use `cta-section-good.png`.

Requirements:
- one clear primary action
- strong but restrained framing
- enough surrounding whitespace
- direct commercial wording

Preferred CTA types:
- Get a quote
- Request pricing
- Speak to an expert
- Calculate cost
- Discuss your workflow

Avoid:
- manipulative urgency
- multiple competing CTA buttons
- desperate sales tone

### Forms
Use `form-simple-good.png` for desktop and `mobile-form-good.png` for mobile.

Requirements:
- simple, useful fields
- clear grouping
- obvious submit action
- clean labels
- minimal friction

Avoid:
- long intimidating forms
- repeated identical forms across the page
- unnecessary required fields
- generic, ugly lead-capture styling

## Mobile rules
Use desktop assets for mood and structure.
Use mobile assets for spacing, stacking, and readability.

On mobile:
- preserve hierarchy before decoration
- follow `mobile-hero-good.png` for first-screen spacing and CTA rhythm
- follow `mobile-services-good.png` for stacked service layouts
- follow `mobile-form-good.png` for form simplicity and CTA/form grouping
- reduce decorative weight before reducing clarity
- keep proof, services, CTA, and forms immediately scannable

## Copy style
Write like a serious logistics or fulfillment business.

The copy should be:
- direct
- practical
- trustworthy
- commercially literate
- easy to scan

Avoid:
- buzzwords
- startup clichés
- “innovation” filler
- emotional fluff

## Decision rules
If uncertain:
1. choose the cleaner option
2. choose the more specific option
3. choose the more businesslike option
4. choose the more trustworthy option
5. choose the option that improves comprehension fastest

- Does the header match `header-clean-business.png` in discipline?

- Do benefit blocks reflect `benefits-kak2c-good.png`?
- Does proof presentation resemble `metrics-proof-good.png`?
